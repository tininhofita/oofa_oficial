# Árvores de Decisão e Templates Contextuais

> Pensamento de design focado no contexto, não em soluções fixas.
> **Guias para DECIDIR, não para copiar.**

---

## 1. Decisão por Público-Alvo

### Para QUEM estamos construindo?

| Público            | Abordagem de Design                                                |
| ------------------ | ------------------------------------------------------------------ |
| **Geração Z**      | Ousado, rápido, mobile-first, cores vibrantes, tipografia massiva. |
| **Millennials**    | Limpo, minimalista, focado em valores, tons pastéis e terrosos.    |
| **Geração X**      | Familiar, confiável, hierarquia tradicional, sem distrações.       |
| **Boomers**        | Alta legibilidade, alto contraste, caminhos lineares simples.      |
| **Enterprise/B2B** | Profissional, focado em dados, eficiência, paleta sóbria.          |

---

## 2. Decisão por Tipo de Produto

### SaaS / Dashboard

- **Prioridade:** Funcionalidade e clareza.
- **Dica:** Use cores de dados semânticas (sucesso/alerta/erro) e grids organizados.

### E-commerce

- **Prioridade:** Confiança e ação.
- **Dica:** Destaque CTAs de compra e use prova social (reviews) visível.

### Landing Page

- **Prioridade:** Storytelling e emoção.
- **Dica:** Foco total no "Hero" e em uma única chamada para ação (CTA).

## 3. Checklist Pré-Design

- [ ] **Público definido?** (Quem vai usar)
- [ ] **Objetivo primário identificado?** (Qual a ação principal)
- [ ] **Estilo perguntado ao usuário?** (Não assuma minimalismo/Bento se ele quiser brutalismo)
- [ ] **Conteúdo disponível?** (Texto real vs Placeholders)
- [ ] **Padrões de mercado seguidos?** (Jakob's Law)
